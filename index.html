Web Bomic Collect

Allows collection of Web Bomic client events via IPad or Android devices, that can be synchronised to any Web Bomic application.

Installing Web Bomic Collect

WB Collect is made of 3 parts that need to be installed separately.

  • A SQL Script that adds a pin number field to the users table (USRS_DAT) in Web Bomic.
  • A Web API interface that fetches and posts data into Web Bomic.
  • A IPad or Android App that allows the collection of the events.

Installing the SQL Script

The SQL script that adds the pin number field to the Web Bomic database is called:-

  • 001 DDL Add Pin Column to Usrs_dat table.sql

Log in to your desired SQL installation using Microsoft SQL server management studio and with sufficient privileges to run the SQL script. Select your desired Web Bomic database. Open the SQL script and then click Execute.

Installing the API

The API is distributed in the form of a zip file named:-

  • WB Collect WEB API.zip

To install the API

  1. Unzip the contents of the API zip file to a directory.
  2. Add an IIS application pool that uses version 4.x of the .NET framework.
  3. Add an IIS application under the IIS Web Site where your Web Bomic application is already installed.
  4. Select the application pool (added in point 2 above) as the application pool to use for the newly added IIS application.
  5. Point the 'Physical Path' of the newly added IIS application to the directory that contains the un-zipped contents.

Installing the App

The application can be installed by scanning the distributed QR image, using a QR scanner app on an IPad or Android device.

NOTE: The application can currently only be installed on a selected number of IPads' due to the app still being in development and not yet fully distributed.

Using WB Collect

First Use

The first time the application is used an any device, the following should be performed.

  1. Log in, using a valid Web Bomic username and pin number.
  2. Click the green sync button. This will populate the lookup lists used in the app, when the app is in offline mode.
